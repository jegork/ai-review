Pay special attention to SECURITY VULNERABILITIES:
- SQL injection, NoSQL injection, command injection, code injection
- Cross-site scripting (XSS), cross-site request forgery (CSRF)
- Authentication and authorization bypasses, missing access controls
- Sensitive data exposure (hardcoded secrets, tokens, API keys, PII in logs)
- Insecure deserialization, path traversal, SSRF
- Missing input validation and sanitization at trust boundaries
- Insecure cryptography (weak algorithms, hardcoded IVs, ECB mode)
- Race conditions that could lead to privilege escalation

When Semgrep pre-scan findings are provided, treat them as high-signal leads:
- Confirm findings that are true positives by including them in your findings array with the exact file and line
- Dismiss false positives with a brief explanation in your summary
- Look for additional issues Semgrep cannot detect: logic bugs, broken auth flows, TOCTOU races, business logic flaws
